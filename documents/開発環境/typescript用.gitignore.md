以下の **.gitignore** は、  
- バックエンド (NestJS + TypeScript + Prisma)  
- フロントエンド (React + TypeScript)  
- Nginx の実行時ファイル  
- PostgreSQL のダンプや履歴ファイル  

をすべてカバーするように作成しました。リポジトリのルートに配置してください。

```gitignore
# -------------------------------------------------
# Node.js / npm / Yarn 関連
# -------------------------------------------------
node_modules/                     # 依存パッケージ全般
npm-debug.log*                    # npm デバッグログ
yarn-debug.log*                   # Yarn デバッグログ
pnpm-debug.log*                   # pnpm デバッグログ

# -------------------------------------------------
# ビルド成果物・テストカバレッジ
# -------------------------------------------------
dist/                             # NestJS のビルド出力 (tsc)
build/                            # React の production ビルド出力
coverage/                         # Jest / テストカバレッジ結果
*.tsbuildinfo                     # TypeScript incremental コンパイル情報

# -------------------------------------------------
# 環境変数ファイル
# -------------------------------------------------
.env                              # 本番・開発環境の実際の設定 (必ず .gitignore)
.env.*                            # .env.local, .env.development など

# -------------------------------------------------
# Prisma 関連
# -------------------------------------------------
prisma/*.sqlite                   # SQLite のローカル DB（開発時に生成されることがある）
prisma/*.db                       # その他のローカル DB ファイル
prisma/dev.db                     # よく使われる名前例
.prisma/                          # Prisma が内部で作成するキャッシュ等
prisma/**/*.log                   # Prisma のログファイル

# -------------------------------------------------
# Nginx (実行時) 関連
# -------------------------------------------------
nginx/logs/                       # アクセス・エラーログ
*.pid                             # PID ファイル
*.sock                            # ソケットファイル

# -------------------------------------------------
# PostgreSQL ダンプ・履歴
# -------------------------------------------------
*.sql                             # SQL ダンプ（手動で作成したバックアップ等）
*.dump
*.backup
pg_dump_*                         # pg_dump 系の自動生成ファイル名例
.pgdump_history                   # psql のダンプ履歴
.pgpass                           # パスワードファイル（ローカル用）

# PostgreSQL クライアントのコマンドヒストリ
.psql_history

# -------------------------------------------------
# フロントエンド (React) 固有
# -------------------------------------------------
frontend/build/                   # CRA のビルド出力（プロジェクト構成に合わせて調整）
frontend/.cache/                  # ビルドツールが生成するキャッシュ
frontend/public/build/            # static ビルド結果が置かれることがある

# -------------------------------------------------
# バックエンド (NestJS) 固有
# -------------------------------------------------
backend/dist/                     # NestJS のビルド出力（上記 dist と同様に除外）
backend/*.log                     # アプリケーションログ等
backend/coverage/                 # テストカバレッジ（バックエンド単体で走らせた場合）
backend/.nyc_output/              # NYC (Istanbul) の一時出力

# -------------------------------------------------
# OS・エディタ固有の不要ファイル
# -------------------------------------------------
.DS_Store                         # macOS の Finder が作るメタデータ
Thumbs.db                         # Windows のサムネイルキャッシュ
.idea/                            # JetBrains 系 IDE 設定ディレクトリ
.vscode/                          # VS Code 設定ディレクトリ
*.sublime-workspace
*.sublime-project

# -------------------------------------------------
# その他一時ファイル・ログ
# -------------------------------------------------
tmp/
temp/
*.log                             # 任意の場所に出力されるログファイル全般
```

**ポイント**

- `node_modules/` はプロジェクトルートでもサブディレクトリでも同様に除外できます。  
- ビルド成果物 (`dist/`, `build/`) とテストカバレッジ (`coverage/`) をまとめて無視しています。  
- 環境変数ファイルは必ず `.gitignore` に入れ、代わりに `.env.example` などをリポジトリに残すと安全です。  
- Prisma のローカル DB（SQLite 等）や内部キャッシュは開発環境だけのものなので除外します。  
- Nginx のログ・PID・ソケットファイル、PostgreSQL のダンプやクライアント履歴も同様に除外しています。  

この `.gitignore` をベースに、プロジェクト固有で追加したいパスがあれば適宜追記してください。
